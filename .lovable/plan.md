

# План перепозиционирования лендинга TokenBel
## От «каталога токенов» к «инвестиционной интеллект-платформе Беларуси»

---

## 1. Executive summary

**Что не так с текущей логикой:**
- Hero уже упоминает токены/акции/облигации, но **визуальный и контентный вес страницы — 80% про токены** (большой блок «Актуальные токены», «Эмитенты токенов», «Новости токенов»).
- Акции и облигации присутствуют только как кнопки в hero — у пользователя **нет ощущения, что это полноценные продуктовые направления**.
- Портфель **не существует как отдельный смысловой блок** — это сильнейшая фича продукта, спрятанная в одной строке преимуществ.
- Хедер не отражает 4 пилларa продукта (есть «Облигации», «Токены», «Вторичный рынок», «Статистика», но нет «Акции» и «Портфель»).
- Блок «Рынок в цифрах» считает только токены (млн BYN/USD/EUR, 242 токена) — нет показателей по акциям/облигациям.
- Блок «Преимущества» рассказывает о фичах, но **не отвечает на вопрос «зачем мне это?»** в разрезе джобов инвестора.
- Нет блока про аналитику эмитентов, которая — реальный moat продукта.
- Нет демонстрации интерфейса (ни одного скриншота дашборда/портфеля/графиков).
- Trust-слой слабый: нет указания на источники данных, частоту обновлений по каждому классу инструментов, методологию.
- Нарратив страницы — «вот токены → вот компании → новости» вместо «вот рынок → вот инструменты → вот ваш портфель → действуй».

**Ядро сдвига позиционирования:**  
TokenBel перестаёт быть «справочником по токенам» и становится **«единым центром принятия инвестиционных решений на белорусском рынке»**: данные + аналитика + личный портфель.

## 2. Новое позиционирование — 3 направления

### Направление А — «Bloomberg для белорусского частного инвестора»
- **Восприятие:** профессиональная market intelligence платформа.
- **Сильные стороны:** статус, повышает воспринимаемую ценность, оправдывает будущую монетизацию.
- **Риски:** отпугнёт новичков (Андрей-персона), завышенные ожидания по depth данных.
- **Когда выбрать:** если ядро ЦА сместится в сторону опытных инвесторов и B2B-клиентов.

### Направление Б — «Один кабинет для всех инвестиций в Беларуси» (агрегатор + портфель)
- **Восприятие:** удобный личный финансовый хаб.
- **Сильные стороны:** прямое попадание в JTBD «не хочу 5 вкладок» — самый частый pain. Портфель становится якорем.
- **Риски:** надо держать паритет данных по всем 3 классам инструментов, иначе «один кабинет» — обман.
- **Когда выбрать:** **сейчас** — наиболее реалистично, опирается на реально существующую фичу портфеля.

### Направление В — «Аналитика и мониторинг рынка ценных бумаг Беларуси»
- **Восприятие:** аналитический инструмент, ближе к research-сервису.
- **Сильные стороны:** оправдывает глубину статистики, вторичный рынок, эмитентов.
- **Риски:** портфель выглядит инородно, конверсия в регистрацию ниже (аналитику читают, не подписываются).
- **Когда выбрать:** для контентного/SEO трека.

### **Рекомендация: Направление Б**, с элементами В в среднем уровне страницы.  
Именно «единый кабинет + портфель» — единственная позиция, где TokenBel объективно лидирует и которую сложно скопировать конкурентам (Finstore/Fainex/Bynex — каждый только про свои токены).

## 3. Messaging architecture

**Primary message (1 строка):**  
«Все токены, акции и облигации Беларуси — в одном кабинете. Со статистикой, аналитикой и вашим личным портфелем».

**Secondary messages:**
1. Полное покрытие рынка — не один эмитент, не одна площадка.
2. Данные обновляются ежеминутно.
3. Портфель считает доходность, налоги, выплаты и сравнивает с рынком.
4. Аналитика эмитентов — финансовое состояние, история выплат, рейтинги.
5. Вторичный рынок — единственное место, где видно реальные сделки между инвесторами.

**Proof points:**
- N эмитентов / N токенов / N акций / N облигаций под наблюдением.
- N платформ-источников (Finstore, Fainex, Bynex…).
- Объём рынка в 3 валютах.
- Частота обновления (ежеминутно для котировок, ежедневно для финотчётности).
- История наблюдений с 2025-06.

**Возражения и ответы:**
| Возражение | Ответ на странице |
|---|---|
| «Это же просто витрина токенов» | Раздел Portfolio + раздел Акции/Облигации с реальной глубиной |
| «Я уже смотрю на Finstore/Fainex напрямую» | Блок «Зачем агрегатор» — сравнение, история, единый портфель поверх всех площадок |
| «Можно ли доверять данным?» | Trust-блок: источники, методология, частота, дисклеймер |
| «Это для профи» | Разделение CTA по уровню: «посмотреть рынок» / «следить» / «вести портфель» |
| «Сколько стоит?» | Явная плашка «Бесплатно» (или ссылка на тариф Standard) |

**Trust signals:** счётчики реальных данных, логотипы площадок-источников, ссылка на FAQ, ссылка на Telegram-канал с историей публикаций, дисклеймер «не является инвестрекомендацией».

**Как уместить 4 пиллара без перегруза:**  
Не давать им равный визуальный вес на одном экране. **Hero — общий зонт; затем один блок-«пиллары» с 4 карточками-входами; затем 2–3 deep-dive блока (Портфель — большой, Аналитика — средний, Вторичный рынок — компактный).** Списки актуальных токенов/акций/облигаций — единая универсальная карусель с табами вместо трёх отдельных секций.

- Отдельная большая секция «Эмитенты токенов» — превращается в часть блока «Аналитика эмитентов».

- 4 кнопки в hero сжимаются до 2 (primary/secondary), остальные точки входа — через блок пилларов.

## 6. Hero — 5 концептов

**Концепт 1: «Единый кабинет»** *(рекомендован)*
- H1: **«Токены, акции и облигации Беларуси — в одном кабинете»**
- Sub: «Актуальные данные по 3 классам инструментов, аналитика эмитентов и личный портфель с расчётом доходности — бесплатно».
- CTA: [Открыть дашборд] [Создать портфель]
- Почему работает: точно описывает что есть, кому, и сразу выводит портфель в hero.

**Концепт 2: «Решение для инвестора»**
- H1: «Принимайте инвестиционные решения на основе данных, а не слухов»
- Sub: «Вся статистика рынка ценных бумаг Беларуси, аналитика эмитентов и ваш портфель в одном месте».
- CTA: [Открыть рынок] [Узнать больше]
- Почему работает: эмоциональнее, апеллирует к pain. Минус: не показывает scope.

**Концепт 3: «Цифры в hero»**
- H1: «X эмитентов. Y инструментов. Один кабинет.»
- Sub: «TokenBel — агрегатор токенов, акций и облигаций Беларуси с личным портфелем».
- CTA: [Смотреть рынок] [Создать портфель]
- Почему работает: сразу даёт proof. Минус: цифры надо держать актуальными.

**Концепт 4: «Job-to-be-done»**
- H1: «Перестаньте вести инвестиции в Excel»
- Sub: «TokenBel автоматически считает доходность, выплаты и налоги по вашим токенам, акциям и облигациям».
- CTA: [Создать портфель] [Посмотреть рынок]
- Почему работает: сильный pain-hook для активных инвесторов. Минус: отсекает новичков.

**Концепт 5: «Категорийный лидер»**
- H1: «Главный агрегатор ценных бумаг Беларуси»
- Sub: «Токены, акции, облигации, аналитика эмитентов и портфель — обновление ежеминутно».
- CTA: [Открыть дашборд] [Telegram]
- Почему работает: сильное позиционирование. Минус: «главный» — претензия, требующая доказательств.

**Рекомендация: Концепт 1.** Прямо отражает позиционирование Б, удобен для SEO, не делает спорных заявлений, явно выводит портфель в hero.

## 7. Архитектура продукта на лендинге

**Не использовать:**
- Tabs в hero — скрывает контент от ботов и мобильных пользователей.
- Comparison grid инструментов между собой — это путает: токены/акции/облигации не альтернативы, а классы.

**Рекомендую гибрид:**
1. **Сразу под hero — блок «4 пиллара» как 4 равные карточки** (Токены, Акции, Облигации, Портфель), каждая с короткой ценностью и счётчиком. Это даёт за 3 секунды понимание scope.
2. **Витрина инструментов — как табы** (Токены / Акции / Облигации) в одной секции с одинаковыми карточками. Это устраняет дублирование 3 секций и держит страницу компактной.
3. **Портфель — отдельная большая секция со скриншотом**, не в табах, не в карточках — как «героическая» фича.

Почему так: 4 карточки задают карту продукта, табы экономят вертикаль страницы, отдельный большой блок Портфеля передаёт его статус как пиллара (а не пункта в списке).

## 8. Стратегия презентации Портфеля

**Где:** сразу после блока «Рынок в цифрах», ДО витрины инструментов. Это «продуктовый» якорь, который должен видеть каждый посетитель выше середины страницы.

**Как объяснять ценность (не общими словами):**
- «Загрузите свои покупки — получите расчёт доходности XIRR по каждому инструменту и портфелю в целом».
- «Прогноз будущих выплат купонов и амортизации на 12 месяцев вперёд».
- «Учёт налогов с купонов автоматически».
- «Сравнение вашей доходности со средней по рынку в той же валюте».
- «Алерты по событиям эмитента: погашение, оферта, просрочка выплаты».

**Какие метрики/расчёты подсветить:**
- Текущая стоимость портфеля по 3 валютам.
- XIRR / средняя доходность к погашению.
- График выплат на 12 месяцев.
- Концентрация по эмитентам (риск).
- Распределение по валютам / срокам / классам инструментов.
- P&L реализованный / нереализованный.

**Дифференциация:**  
Площадки (Finstore/Fainex/Bynex) считают портфель **только из своих** токенов. TokenBel считает **сквозной** портфель из инструментов всех площадок + акции + облигации. Это **единственное место** в Беларуси, где это возможно. Этот тезис должен быть в блоке буквально.

**Визуал:** скриншот реального дашборда портфеля (или достоверный мокап) — обязателен. Без него блок не работает. Минимум: график стоимости + список позиций + календарь выплат.

**Как избежать generic-звучания:** не использовать слова «удобно», «просто», «всё в одном». Заменить на конкретные метрики и цифры («XIRR», «12 месяцев выплат», «3 валюты», «N площадок») и скриншот.

## 9. Информационная иерархия — что куда

**Поднять выше:**
- Портфель (сейчас — строчка в преимуществах → отдельный большой блок выше витрины).
- Акции и облигации (сейчас — кнопки → пилларная карточка + таб в витрине).
- Метрики покрытия рынка (сейчас глубже → расширить в «Рынок в цифрах»).

**Опустить вниз:**
- Новости / лента — после trust-блока.
- Большой список эмитентов — заменить на 1 блок «Аналитика эмитентов» со скриншотом + ссылкой.

**Уменьшить:**
- Витрину токенов: с 6+ карточек до 3–4 в табе «Токены», с явной кнопкой «все токены».
- 4 кнопки в hero → 2 CTA.

**Удалить:**
- Дублирующиеся CTA «Подробнее» в каждой карточке там, где вся карточка кликабельна.
- «Удобный поиск» / «Всё в одном месте» (уже сделано).
- Декоративные блоки без продуктовой нагрузки.

**Добавить:**
- Блок Портфеля со скриншотом.
- Блок «4 пиллара».
- Блок «Аналитика эмитентов».
- Блок «Зачем агрегатор» (vs прямые площадки).
- Trust-блок с логотипами источников и методологией.
- Скриншоты интерфейса (минимум 2: портфель + карточка эмитента).

## 10. Контентные пробелы

| Тема | Чего не хватает | Что добавить |
|---|---|---|
| **Акции** | Никакого описания, кроме кнопки | Карточка-пиллар + таб в витрине + 1 строка про источник данных (биржа/реестр) + счётчик |
| **Облигации** | То же | Карточка-пиллар + таб + типы (корп./гос.) + счётчик |
| **Портфель** | Только упоминание | Полноценный блок (см. §8) + скриншот + список метрик |
| **Аналитика** | Не объяснено, что именно анализируется | Блок «Аналитика эмитентов» с перечнем: финотчётность, выплатная дисциплина, рейтинги, динамика объёмов |
| **Эмитенты (intelligence)** | Только список логотипов | Скриншот карточки эмитента с реальными метриками |
| **Trust** | Только дисклеймер | Логотипы источников (Finstore, Fainex, Bynex и др.), методология обновлений, частота, дата последней синхронизации, ссылка на FAQ |
| **Категории инвесторов (с 01.04.2026)** | Есть только в FAQ | Краткая плашка «Готовы к новому регулированию» со ссылкой на FAQ |
| **Цена / тарифы** | Не упоминается | Бейдж «Бесплатно» в hero + ссылка на /pricing/ |

## 11. Conversion strategy — система CTA

**Иерархия:**
- **Primary CTA (везде):** «Открыть дашборд» — самое низкое сопротивление, ведёт на dashboard.tokenbel.info.
- **Secondary CTA (везде):** «Создать портфель» — продуктовый якорь, выше воронка вовлечения.
- **Exploratory CTA (контекстный):** «Подписаться в Telegram», «RSS», «Все токены/акции/облигации».

**Маппинг по уровню намерения:**
| Намерение | CTA | Где разместить |
|---|---|---|
| Просто исследует | «Посмотреть рынок» / «Статистика» | Hero (secondary), Рынок в цифрах |
| Сравнивает инструменты | «Все токены / акции / облигации» | Витрина с табами |
| Готов работать с дашбордом | «Открыть дашборд» / «Войти» | Hero (primary), хедер |
| Интересуется портфелем | «Создать портфель» | Блок Портфеля, финальный CTA |
| Хочет алерты/мониторинг | «Подписаться в Telegram» / «RSS» | Trust-блок, футер |
| Эмитент | «Связаться» | Блок аналитики эмитентов |

**Правило:** на одной высоте экрана — не больше 2 CTA. Иначе размывается фокус.

## 12. Trust & credibility

**Что добавить:**
- **Покрытие рынка:** «N эмитентов · N токенов · N акций · N облигаций · N площадок».
- **Частота обновлений:** «Котировки — ежеминутно. Финотчётность — ежедневно. Сделки вторичного рынка — в реальном времени» (только то, что правда).
- **Источники данных:** логотипы Finstore, Fainex, Bynex и др. + строчка «Данные собираются из официальных API площадок».
- **Глубина аналитики:** «Финансовая отчётность с YYYY года», «История выплат по каждому эмитенту».
- **Методология:** ссылка на отдельную страницу /methodology/ или раздел FAQ — как считаются доходности, откуда берутся данные.
- **Социальные доказательства:** Telegram-канал с N подписчиками + дата создания (если число достойное).
- **Дисклеймер:** видимый, но не пугающий — «информационный сервис, не инвестрекомендации».

**Чего избегать:** фейковых отзывов, абстрактных «нам доверяют 1000+ инвесторов» без подтверждения, «лучший в Беларуси» без атрибуции.

## 15. Mobile-first

- **Hero:** 2 CTA в столбик, метрики — горизонтальная скроллируемая лента или 2×2 сетка.
- **Пиллары:** 2×2 на мобильном (а не 1 в столбик — иначе занимает целый экран).
- **Рынок в цифрах:** свайпаемые карточки, не сетка 3×N.
- **Портфель:** скриншот — на всю ширину, без обрезки; список метрик — раскрывающийся аккордеон.
- **Витрина с табами:** табы — горизонтальный скролл; карточки — 1 в строку.
- **Сравнительная таблица «зачем агрегатор»:** на мобильном — превратить в 3 коротких карточки.
- **Скрыть/отложить:** новости (lazy-load ниже фолда), детальная аналитика эмитентов (вместо неё — короткая карточка + ссылка).
- **Sticky CTA снизу** на мобильном: «Открыть дашборд» — увеличит конверсию.

## 16. Идеальный нарратив страницы

> 1. **Что это** → Hero: «Токены, акции, облигации Беларуси — в одном кабинете».  
> 2. **Что внутри** → 4 пиллара: ага, есть всё.  
> 3. **Доказательство охвата** → Рынок в цифрах: они правда видят весь рынок.  
> 4. **Что я получу лично** → Портфель: я смогу управлять своими деньгами здесь.  
> 5. **Прямо сейчас** → Витрина с табами: вот что доступно сегодня.  
> 6. **Глубина** → Аналитика эмитентов + Вторичный рынок: это не просто список.  
> 7. **Почему не напрямую** → Зачем агрегатор: один портфель поверх всех площадок.  
> 8. **Можно доверять** → Trust: источники, методология, частота.  
> 9. **Действие** → Финальный CTA: создать портфель.  

Поток: **что → объём → польза для меня → доказательства → действие.**

## 18. A/B-гипотезы (≥10)

1. **Hero H1:** «В одном кабинете» (Концепт 1) vs «Перестаньте вести в Excel» (Концепт 4) — конверсия в регистрацию.
2. **Primary CTA hero:** «Открыть дашборд» vs «Создать портфель» — клики по primary.
3. **Порядок блоков:** Портфель ДО витрины vs ПОСЛЕ витрины — глубина скролла + клики на «Создать портфель».
4. **Витрина:** табы (токены/акции/облигации) vs 3 раздельные секции — время на странице.
5. **Пиллары:** 4 карточки vs 2 карточки + 2 строки — CTR на пилларные ссылки.
6. **Скриншот портфеля:** реальный скриншот vs упрощённый мокап vs без визуала — конверсия в портфель.
7. **Hero CTA:** 2 кнопки vs 3 кнопки vs 1 + текстовая ссылка.
8. **Trust-блок:** логотипы площадок наверху страницы (под hero) vs внизу — bounce rate.
9. **Sticky mobile CTA:** есть/нет — конверсия на мобильном.
10. **Категоризация инструментов:** «Токены / Акции / Облигации» vs «По доходности / По валюте / По сроку».
11. **Блок «Зачем агрегатор»:** сравнительная таблица vs 3 карточки-сравнения.
12. **Бейдж «Бесплатно»:** в hero vs в финальном CTA vs в обоих местах.
13. **Цифры в hero:** «X эмитентов · Y инструментов» vs без цифр.
14. **Социальное доказательство:** счётчик подписчиков Telegram в hero vs в trust-блоке vs нигде.
15. **Имя пиллара:** «Портфель» vs «Личный кабинет инвестора» vs «Аналитика портфеля» — CTR.
